## Hugging Face

Handles anything that is repetitive, broad, or exploratory. Runs locally or cheaply on PikaPods with no per-token billing.

NEXT.JS SYSTEM PROMPT EXAMPLE
You are a senior Next.js developer. Stack: App Router, TypeScript, Tailwind CSS v3, shadcn/ui, Supabase, Vercel, pnpm.

- Always cite file path and line number
- Prefer server components unless interactivity is required
- Flag any "any" types and suggest the correct TypeScript interface
- Flag missing Supabase RLS policies on mutating routes

- Keep responses concise: problem → location → fix

- Use a well-instructed system prompt — define role, output format, and expectations. The more specific you are, the less Claude-level reasoning you need.

- Seed with examples — paste one or two examples of the output format at the start of your prompt. HF models follow patterns well; one good example cuts vague responses significantly.
- Iterate the system prompt, not the question — if output is inconsistent, refine the system prompt rather than rephrasing each time. A tuned system prompt compounds across every future query.

### Local Setup

The fastest way to get started — runs entirely on your machine, no accounts needed.

**Step 1 — Clone your repo**

```bash
git clone https://github.com/your-org/your-repo.git
cd your-repo
```

**Step 2 — Install Ollama and pull the coding model**

Download [Ollama](https://ollama.com) for Mac, Windows, or Linux, then pull the model:

```bash
ollama pull qwen2.5-coder:7b
ollama run qwen2.5-coder:7b
```

Ollama serves the model locally at `http://localhost:11434` — no API key, no usage limits.

**Step 3 — Install the Continue extension in VS Code**

Open VS Code → Extensions (`Cmd+Shift+X`) → search **Continue** → install **Continue** by Continue.

**Step 4 — Point Continue at Ollama**

Continue auto-detects Ollama on first launch. To configure manually, open `.continue/config.json` in your project root:

```json
{
  "models": [
    {
      "title": "Qwen2.5 Coder (local)",
      "provider": "ollama",
      "model": "qwen2.5-coder:7b"
    }
  ]
}
```

**Step 5 — Continue indexes your codebase automatically**

On first open, Continue walks your repo and builds a local index. From there, `Cmd+I` opens the inline editor and `Cmd+L` opens the chat sidebar — both have full codebase context without you pasting anything.

**Step 6 — If your device can't run the model locally**

Create a [Hugging Face Inference Endpoint](https://ui.endpoints.huggingface.co) for `Qwen/Qwen2.5-Coder-7B-Instruct` and update your Continue config to point at it instead:

```json
{
  "models": [
    {
      "title": "Qwen2.5 Coder (HF endpoint)",
      "provider": "openai",
      "model": "Qwen/Qwen2.5-Coder-7B-Instruct",
      "apiBase": "https://your-endpoint.huggingface.cloud/v1/",
      "apiKey": "hf_your_token_here"
    }
  ]
}
```

The endpoint runs on HF hardware and bills only for active inference time — typically under $9/mo for bulk coding work.

## Claude Code

Handles anything that requires deep reasoning, multi-file context, or precision — refactors, bug diagnosis, and migrations where a wrong move costs real time.

2. Authenticate: run "claude" in your terminal and follow the login steps
3. Optional — set ANTHROPIC_API_KEY in your shell profile for API access

VS CODE EXTENSION
1. Open VS Code → Extensions (Cmd+Shift+X)
2. Search "Claude Code" → Install "Claude Code" by Anthropic
</AiPrompt>

**Getting the most out of Claude Projects:**

- **Create a project per codebase** — Claude Projects retain context across conversations. Set one up for each repo so Claude already knows your stack, conventions, and architecture before you type a word.
- **Add a project instructions file** — paste your tech stack, folder structure, naming conventions, and any rules ("never use `any`, always validate with Zod") into the project instructions. Claude applies them to every session automatically.
- **Upload key files as project knowledge** — add your `README.md`, `schema.sql`, and key config files. Claude references them without you needing to paste context each time.
- **Reserve Projects for precision work** — use the project session for refactors, migrations, and bug diagnosis where full codebase context matters. Route the rest to Hugging Face to preserve your API budget for these high-value sessions.

## Cost Comparison

<HybridAiSavings />

The math is straightforward: Hugging Face on PikaPods runs around $9/month for a capable inference endpoint, Supabase and Vercel both run free on their starter tiers. That same budget covers the bulk tasks that would otherwise push your Claude API bill into $60–$120+ territory. Claude stays reserved for the work that actually benefits from its reasoning depth.
